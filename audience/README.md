# audience

**Definition:** The intended readership or age group for which the work is primarily designed or most appropriate.

---

## What is Audience?

Audience answers: **who is this book for?** It is a property of the work's design and intent — not of any individual reader. A 40-year-old can read a Young Adult novel; that doesn't change the work's audience tag.

Audience is distinct from every other type:
- It is not a genre (Horror can be YA or Adult)
- It is not a format (a Graphic Novel can be for Children or Adults)
- It is not a subject (a book about childhood is not necessarily *for* children)

---

## Core Rules

**Do use audience to capture:**
- The primary intended readership by age or expertise level
- The audience the publisher, author, and librarian would classify the work for
- The level of assumed background knowledge or reading maturity

**Do NOT use audience to describe:**
- Who *can* read the book — almost any book can be read by anyone
- The age of characters in the book (a book about a 12-year-old is not necessarily Middle Grade)
- The subject matter's complexity alone (a simple-writing adult book is still Adult)

**The Design Intent Rule:** Ask "who did the author/publisher design this for?" not "who is capable of reading this?"

**The Content Rule (for age tiers):** If a work contains mature themes (explicit sex, graphic violence, adult trauma) that would be inappropriate for children, it is Adult regardless of the reading level.

---

## Vocabulary Notes

- **Juvenile** is the broad catch-all term for under-18 content when more specific age data is unavailable. It maps to the historical OL `subject` of "Juvenile fiction" / "Juvenile literature" and to Thema's Y category. Prefer a more specific age tier when known.
- **Academic** and **Professional/Technical** describe expertise-level targeting, not age.
- A work can carry more than one audience tag when it genuinely bridges groups (e.g., a picture book designed for parent-child reading together might be `Children` + `General / All Ages`), but this should be used sparingly.

---

## Relationship to OL Data

This type replaces and normalizes OL legacy subjects like:
- "Juvenile fiction" → `Juvenile`
- "Children's fiction" → `Children`
- "Young adult fiction" → `Young Adult`
- "Readers (Adult)" → `Adult`
- "Academic" works → `Academic`
